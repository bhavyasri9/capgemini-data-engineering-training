# Phase 3A – Data Quality & Cleaning

## Objective

To understand the importance of data cleaning in data engineering. The messy dataset contained multiple issues such as null values, duplicates, and invalid data, which needed to be cleaned before performing any analysis.

## What I Did

* Identified data quality issues such as missing values, duplicate records, and invalid age values
* Cleaned the dataset by removing null records and duplicates
* Filtered out invalid data (negative age values)
* Validated the cleaning process by comparing row counts before and after cleaning
* Performed aggregation to find the number of customers in each city

## Key Learnings

* Real-world datasets are often messy and require proper cleaning
* Ignoring data quality can lead to incorrect results
* Data validation is an important step in any pipeline
* Cleaning should always be done before performing transformations or analysis

## Challenges Faced

* Identifying all types of data issues in the dataset
* Ensuring no valid data was removed during cleaning
* Understanding the sequence of cleaning steps

## Conclusion

This phase helped me understand that data quality is a critical part of data engineering. Cleaning and validating data ensures accurate results and reliable insights for downstream processing.
