# Phase 3 – ETL Pipeline & Data Quality

## Objective

To understand how data engineering workflows are built using ETL (Extract, Transform, Load) principles and to handle data quality issues effectively.

## What I Implemented

* Built an ETL pipeline using PySpark
* Loaded data from CSV files
* Cleaned data by handling null values, duplicates, and invalid records
* Performed business transformations like daily sales and revenue calculations
* Generated a final reporting dataset
* Saved processed output

## Key Learnings

* Real-world data is often messy and requires cleaning
* Data validation is essential before processing
* ETL pipelines follow a structured flow
* PySpark is powerful for handling large datasets
* Combining multiple transformations creates meaningful insights

## Challenges Faced

* Handling null values correctly
* Understanding the sequence of ETL steps
* Debugging errors during transformations

## Conclusion

This phase helped me understand how data engineers build pipelines to process raw data into structured insights. It strengthened my ability to clean, transform, and manage data efficiently.
