#Phase1 – SQL to PySpark

## Objective

To learn basic SQL queries and convert them into PySpark DataFrame operations.

## queries Completed

* Displaying all records from a dataset
* Filtering data based on conditions
* Selecting specific columns
* Applying conditions like age filters
* Performing basic aggregation using groupBy

## SQL to PySpark Mapping

I learned how common SQL operations map to PySpark:

* SELECT → select()
* WHERE → filter()
* GROUP BY → groupBy()
* COUNT → count()

This helped me understand how similar the logic is, even though the syntax differs.

## Key Learnings

* PySpark DataFrames behave like structured tables
* Transformations are applied step-by-step
* Writing PySpark code requires thinking in terms of operations on datasets rather than direct queries
  
## Challenges
- Understanding PySpark syntax
- Converting SQL logic into DataFrame operations

## How I Solved Them

* Practiced multiple examples
* Compared SQL queries with PySpark

## Conclusion

This phase gave me a solid understanding of how SQL concepts are applied in PySpark. It also helped me build confidence in writing basic data transformations, which will be useful for more advanced topics in the next phases.
