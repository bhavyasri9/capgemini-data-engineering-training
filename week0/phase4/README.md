# Phase 4 – Business Pipeline & Analytics

## Objective

The objective of this phase was to build a complete end-to-end data pipeline that generates meaningful business insights. This phase focuses on applying ETL concepts along with analytics to simulate real-world data engineering workflows.

## What I Implemented

* Built an ETL pipeline using PySpark
* Loaded and cleaned datasets by handling null values and data types
* Performed multiple business transformations :
  * Daily sales calculation
  * City-wise revenue analysis
  * Identifying top customers
  * Detecting repeat customers
  * Customer segmentation based on spending
* Created a final reporting dataset combining all insights
* Saved the output for further use

## Key Learnings

* Learned how to design end-to-end data pipelines
* Understood how business logic is applied to data
* Gained experience in combining multiple transformations

## Challenges Faced

* Managing multiple transformations efficiently
* Ensuring correct joins between datasets
* Applying segmentation logic accurately

## Conclusion

This phase helped me move from writing individual queries to building a complete data pipeline. It strengthened my understanding of how raw data is transformed into actionable insights, which is a key responsibility of a data engineer.
