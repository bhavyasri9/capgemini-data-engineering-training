# Day 3 – Advanced SQL: CASE WHEN & Window Functions

## Overview

On Day 3 of the training, I worked on advanced SQL concepts including CASE WHEN logic, nested conditions, and window functions. This helped me understand how to apply business rules and perform analytical operations on data.

## What I Learned

- Writing conditional logic using CASE WHEN
- Handling complex conditions using nested CASE statements
- Applying real-world business rules on datasets
- Understanding and using window functions:
  - ROW_NUMBER()
  - RANK()
  - DENSE_RANK()

## Practical Tasks Performed

- Solved multiple real-world scenarios using CASE WHEN:
  - Customer segmentation based on usage and billing
  - Plan upgrade suggestions
  - Offer eligibility and reward classification
  - Usage pattern analysis

- Implemented window functions to:
  - Assign row numbers to records
  - Rank employees based on salary
  - Perform partition-based ranking (department-wise ranking)

## Key Concepts Practiced

- CASE WHEN (conditional transformations)
- Nested CASE statements
- Business logic implementation in SQL
- Window functions
- Partitioning and ranking data

## Challenges Faced

- Writing complex nested CASE conditions
- Understanding differences between ROW_NUMBER, RANK, and DENSE_RANK
- Applying window functions with correct partitioning
- Debugging logical errors in conditions

## Conclusion

Day 3 helped me move from basic SQL queries to advanced data transformation and analytical techniques. I learned how to apply business logic using CASE statements and perform ranking using window functions, which are essential for real-world data engineering tasks.
